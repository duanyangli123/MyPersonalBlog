# 个人博客网站 — AI 协作工作流

## 一、AI 模型使用指南

### 豆包 (Volcengine) — 中文内容引擎
**适用场景：**
- 博客文章撰写与润色
- 关键词提取与优化
- 内容摘要自动生成
- 中文语法检查

**配置：**
- 环境变量：`DOUBAO_API_KEY`
- API 端点：`https://ark.cn-beijing.volces.com/api/v3`
- 模型 ID：在火山方舟控制台创建推理接入点获取

### DeepSeek V4 Pro — 代码/SEO 引擎
**适用场景：**
- SEO 标题/描述生成
- 代码示例生成
- 技术文档编写

**配置：**
- 环境变量：`DEEPSEEK_API_KEY`
- API 端点：`https://api.deepseek.com/v1`

### Xiaomi MiMo V2 — 创意内容引擎
**适用场景：**
- 文章配图描述生成
- 社交媒体推广文案
- 创意标题建议
- 文章风格分析

**调用示例：**
`
为这篇技术文章生成 3 个社交媒体推广文案：
标题：Next.js 14 App Router 完全指南
风格：专业但不枯燥
`

### Xiaomi MiMo V2 Pro — 高级分析引擎
**适用场景：**
- 文章推荐算法
- 用户行为分析
- 内容质量评估
- 性能优化建议
- SEO 策略制定

**调用示例：**
`
分析以下文章的 SEO 质量，并给出优化建议：
[文章内容]
`

## 二、OpenCode Agent 工作流

### 工作流程

`
1. 需求分析
   └── 用户描述需求 → Agent 理解并拆解任务

2. 方案设计
   └── Agent 分析代码库 → 设计实现方案

3. 代码实现
   └── Agent 编写代码 → 自动测试 → 验证

4. 代码审查
   └── Agent 检查代码质量 → 优化建议

5. 部署验证
   └── Agent 构建项目 → 验证功能
`

### Agent 指令模板

**创建组件：**
`
请创建一个 PostCard 组件，用于展示博客文章卡片。
要求：
- 显示标题、摘要、日期、标签
- 支持暗色模式
- 响应式设计
- 使用 TypeScript
`

**修复 Bug：**
`
在文章详情页，代码块的复制按钮点击后没有反应。
请排查并修复这个问题。
`

**代码重构：**
`
请重构 src/components/blog/ 目录下的组件，
将重复的样式提取到公共样式文件中。
`

## 三、AI 辅助开发流程

### 阶段一：需求分析
`
1. GLM 5.1 分析需求文档
2. 生成技术方案初稿
3. 识别关键技术点
`

### 阶段二：设计阶段
`
1. DeepSeek V4 Pro 设计 API 接口
2. 设计数据库 Schema
3. 设计组件架构
`

### 阶段三：开发阶段
`
1. OpenCode 实现核心组件
2. DeepSeek V4 Pro 生成工具函数
3. 实时测试验证
`

### 阶段四：内容生成
`
1. GLM 5.1 生成博客文章
2. MiMo V2 生成配图描述
3. MiMo V2 Pro 优化 SEO
`

### 阶段五：测试优化
`
1. DeepSeek V4 Pro 生成测试用例
2. MiMo V2 Pro 分析性能瓶颈
3. OpenCode 执行优化
`

## 四、Prompt 模板库

### 文章生成
`
角色：你是一位资深技术博主
任务：撰写一篇关于 [主题] 的技术文章
要求：
- 目标读者：[读者描述]
- 文章长度：[字数] 字
- 内容结构：引言 → 正文 → 总结
- 包含代码示例
- SEO 友好
`

### 代码生成
`
角色：你是一位 Next.js 专家
任务：实现 [功能描述]
技术栈：
- Next.js 14 App Router
- TypeScript
- Tailwind CSS
要求：
- 使用 server components
- 完整的错误处理
- 响应式设计
- TypeScript 类型完整
`

### SEO 优化
`
角色：你是一位 SEO 专家
任务：优化以下文章的 SEO
文章标题：[标题]
文章内容：[内容]
请提供：
1. 优化后的标题（60字符内）
2. Meta 描述（160字符内）
3. 推荐关键词（5-10个）
4. 结构化数据建议
`

## 五、内容生成流程

### 博客文章生成
`
1. 确定文章主题和大纲
2. GLM 5.1 生成初稿
3. 人工审核和修改
4. MiMo V2 生成配图描述
5. MiMo V2 Pro 优化 SEO
6. DeepSeek V4 Pro 生成代码示例
7. 最终发布
`

### 系列文章规划
`
1. MiMo V2 Pro 分析热门话题
2. GLM 5.1 规划系列大纲
3. 按计划逐步生成内容
4. 保持风格一致性
`

## 六、代码审查与优化

### AI 辅助审查
`
1. DeepSeek V4 Pro 分析代码质量
2. MiMo V2 Pro 识别性能问题
3. OpenCode 执行优化
4. 人工验证结果
`

### 优化优先级
1. 性能瓶颈（高）
2. 安全漏洞（高）
3. 代码重复（中）
4. 可读性（中）
5. 命名规范（低）

## 七、成本控制

### API 调用策略
- 缓存 AI 生成结果
- 批量处理相似请求
- 避免重复调用
- 使用合适的模型（简单任务用小模型）

### 日志记录
- 记录每次调用的 token 数
- 统计月度成本
- 监控异常调用
